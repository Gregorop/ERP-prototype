п 2.3.2 варианты оптимизации запроса
1. Считать родительскую категорию при каждом запросе это долго.
Оптимальнее ввести столбик root_id и заполнять его тригером.
Категории добавляются явно реже, поэтому
небольшая разовая задержка на расчет при добавлении лучше чем регулярная задержка при чтении.

2. Сделать материализованное представление
Топ-5 товаров за месяц явно не меняется каждые полчаса, ничего страшного обновлять эту инфу раз в день.

3. Сделать партицирование по месяцам, чтобы уменьшить seqscan
